Two complete, runnable reference tools live in the repo. Each does the **full LTI loop** —
SSO launch → deep-link content selection → grade passback (AGS) → roster (NRPS) — so you can see
LTIkit working against a real LMS before wiring your own app.

<Aside type="tip">
New here? Start with **next-prisma-demo** — SQLite means **zero external services** (no Docker, no
Supabase account), so it's the fastest path from clone to a working launch.
</Aside>

## Run it in ~2 minutes (next-prisma-demo)

```bash
git clone https://github.com/alasim/ltikit
cd ltikit
pnpm install

cd examples/next-prisma-demo
cp .env.example .env.local     # then fill APP_URL, AUTH_SECRET, and the keypair
pnpm db:migrate                # SQLite → dev.db (Prisma + better-sqlite3)
pnpm db:seed                   # demo login: demo@ltikit.dev / ltikit-demo
pnpm dev
```

Visit `/` → prompted to `/login` (the seeded demo account). Launched from an LMS → skips login
entirely; the launch handler mints the session. To register with an LMS, point its LTI 1.3
auto-config at `APP_URL/api/lti/register` (Dynamic Registration) — no manual platform insert.

Need a public HTTPS URL for the LMS to reach the tool (its iframe + `SameSite=None; Secure`
cookies require it) — a dev tunnel works for local testing.

## Which demo shows what

| Capability | next-prisma-demo | next-demo |
|---|---|---|
| Storage | Prisma / SQLite (zero external service) | Supabase / Postgres |
| Framework binding | `@ltikit/next` | `@ltikit/next` |
| Auth / session | NextAuth v5 + seeded login page | app-owned session |
| SSO launch · deep linking · AGS · NRPS | ✅ | ✅ |
| Dynamic Registration | ✅ | — |
| Cookieless Platform Storage | ✅ | — |
